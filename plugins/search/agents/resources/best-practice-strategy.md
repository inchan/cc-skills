# 모범 사례 검색 전략

실행 가능한 코드 샘플, 검증된 디자인 패턴, 프로덕션 레벨 구현 예제를 찾는 전략입니다.

---

## 목적

- 특정 기능의 실제 구현 코드 제공
- 공식적으로 권장하는 패턴과 구조
- 프로덕션 환경에서 검증된 솔루션
- 샘플 프로젝트, 레퍼런스 코드

---

## Tier 필터

**우선순위**:
1. Tier 1-2 (공식 예제) - 최우선
2. Tier 3 (커뮤니티 검증 필요) - 허용
3. Tier 4 - 제한적 허용 (유명 개발자만)

---

## 검색 키워드 전략

### 코드 중심 키워드

- "example", "tutorial", "sample"
- "getting started", "step by step"
- "real world", "production"
- "best practice", "design pattern"

### GitHub 우선 검색

```
"{tech} official example {feature}"
site:github.com/{org}/{repo}/examples
```

---

## 우선순위 검색 전략

### 1. 공식 샘플/예제 (최우선, 50% 비중)

```
검색 대상:
- GitHub 공식 저장소: {org}/{repo}/tree/main/examples
- 공식 문서 Examples 섹션
- Context7 MCP: get-library-docs(mode='code')
- 공식 튜토리얼: Getting Started, Guides
```

### 2. 검증된 레퍼런스 프로젝트 (30% 비중)

```
검색 대상:
- Awesome List: awesome-{tech}
- Real World App: realworld.io
- 공식 추천 프로젝트: Built with {framework}
```

### 3. 유명 오픈소스 프로젝트 (15% 비중)

```
검색 대상:
- GitHub Stars 1000+ 프로젝트
- Vercel, Netlify 등 플랫폼 예제
- 유명 기업 오픈소스 (Google, Facebook, Airbnb)
```

### 4. 기술 블로그 실전 가이드 (5% 비중)

```
검색 대상:
- 공식 블로그: blog.{tech}.dev
- 유명 개발자: Kent C. Dodds, Dan Abramov 등
- 플랫폼 블로그: Vercel, Netlify 등
```

## 코드 품질 기준

| 기준 | 필수 | 가산점 |
|------|------|--------|
| 공식 저장소 | ✅ | +10 |
| 테스트 코드 포함 | ✅ | +8 |
| 최신 업데이트 (6개월) | ✅ | +5 |
| GitHub Stars 1000+ | - | +5 |
| 완전한 예제 (실행 가능) | ✅ | +10 |
| 문서화 충실 | - | +3 |
| TypeScript 지원 | - | +2 |

### 제외 기준

- ❌ 오래된 코드 (2년 이상 미업데이트)
- ❌ 불완전한 스니펫 (import 없음, 컨텍스트 부족)
- ❌ 프로덕션 부적합 (보안 이슈, 하드코딩)
- ❌ 비공식 실험적 코드 (alpha, beta)

## 품질 점수 계산

```
점수 = 공식성(0-10) + 완전성(0-10) + 최신성(0-5) + 인기도(0-5) + 품질(0-10)

30점 이상: 강력 추천
20-29점: 추천
10-19점: 참고
10점 미만: 제외
```

## 정보 구조화

### 샘플 코드 분석

```
FOR EACH 샘플 코드:
  1. 코드 목적 파악
  2. 주요 패턴 추출
  3. 의존성 확인
  4. 실행 방법 정리
  5. 주의사항 도출
```

### 모범 사례 추출

**공통 패턴 식별**:
- 여러 샘플에서 반복되는 구조
- 공식 문서에서 강조하는 패턴
- 커뮤니티에서 널리 사용되는 패턴

**안티패턴 식별**:
- 피해야 할 패턴
- 보안 취약점
- 성능 이슈

## 품질 기준

- **품질 점수**: 30점 이상만
- **코드 포함 여부**: 필수
- **GitHub Stars**: 참고 (1000+ 가산점)
- **최대 결과 수**: 10개

## 출력 권장

### 요약 형식

```markdown
## 추천 샘플 코드
- [{샘플명}]({URL}) - {출처} (공식/검증)

## 핵심 패턴
1. {패턴} - {설명}

## 빠른 예제
```{language}
{코드}
```
```

```markdown
## 초기 검색 결과 (Top 5)

## 더 알아보고 싶은 영역
- [ ] 공식 예제 상세 보기
- [ ] 프로덕션 가이드
```
