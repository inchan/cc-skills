# 문서 작성 가이드라인

> 프로젝트 문서 작성 및 관리 원칙

---

## 0. 핵심 원칙

**단일 진실 원천 (Single Source of Truth)**
- 하나의 정보는 하나의 위치에만 존재
- 중복 최소화, 참조 최대화

---

## 1. 문서 중복 최소화

### [Rule 1.1] 중복 금지 원칙

**정의**: 동일하거나 유사한 정보가 2개 이상의 파일에 존재하는 경우

**허용되지 않는 중복**:
- ❌ 동일한 설치 가이드가 README.md와 QUICK_START.md에 존재
- ❌ 프로젝트 구조 설명이 여러 문서에 반복
- ❌ 동일한 코드 예제가 다른 문서에 복사

**예외 (허용되는 중복)**:
- ✅ 요약 vs 상세 (README에 요약, 별도 문서에 상세)
- ✅ 다른 관점 (사용자 가이드 vs 개발자 가이드)
- ✅ 의도적 강조 (중요 경고를 여러 곳에 반복)

### [Rule 1.2] 중복 제거 전략

**우선순위**:
1. **참조 사용**: 링크로 다른 문서 참조
2. **계층 분리**: 개요 문서 + 상세 문서 분리
3. **포함 제거**: 완전히 겹치는 문서 중 하나 삭제

**예시**:
```markdown
<!-- ❌ Bad: 중복 작성 -->
## 설치 방법
1. git clone ...
2. npm install ...

<!-- ✅ Good: 참조 -->
## 설치 방법
[설치 가이드](./INSTALL.md) 참고
```

### [Rule 1.3] 중복 검증 체크리스트

문서 작성/수정 시 확인:
- [ ] 이 정보가 다른 문서에 이미 존재하는가?
- [ ] 존재한다면, 참조로 대체 가능한가?
- [ ] 두 문서가 80% 이상 겹치는가? (겹치면 통합)
- [ ] 향후 정보 변경 시 여러 곳을 수정해야 하는가?

## 2. 문서 역할 분리

### [Rule 2.1] 문서별 명확한 역할 정의

| 문서 | 역할 | 대상 | 길이 |
|------|------|------|------|
| **README.md** | 프로젝트 진입점, 전체 개요 | 외부 방문자 | 짧음 (1-2 페이지) |
| **QUICK_START.md** | 빠른 시작 가이드 | 신규 개발자 | 중간 (2-3 페이지) |
| **docs/instruction.md** | 원본 지시사항 기록 | 내부 참조 | 가변 |
| **docs/requirements.md** | 프로젝트 요구사항 | 개발자 | 중간 |
| **docs/workflows.md** | 작업 흐름 정의 | 개발자 | 중간 |
| **docs/guidelines/*.md** | 구현 가이드라인 | 개발자 | 길이 무관 |

### [Rule 2.2] 역할 충돌 해결

**상황**: README.md와 QUICK_START.md가 80% 이상 겹침

**해결 방안**:

**옵션 A - README 중심** (권장):
```
README.md: 프로젝트 소개 + 빠른 시작 (통합)
QUICK_START.md: 삭제
```

**옵션 B - 역할 분리**:
```
README.md: 프로젝트 소개, 설치만 (외부용)
QUICK_START.md: 상세 개발 가이드 (내부용)
```

**선택 기준**:
- 프로젝트 크기 작음 → 옵션 A
- 프로젝트 크기 큼 (100+ 파일) → 옵션 B

## 3. 문서 구조 원칙

### [Rule 3.1] 계층적 문서 구조

```
README.md (최상위 진입점)
  ↓ 참조
docs/
├── instruction.md (원본 지시)
├── requirements.md (요구사항)
├── workflows.md (작업 흐름)
└── guidelines/
    ├── development.md (개발 가이드)
    ├── documentation.md (문서 가이드)
    └── tool-creation.md (도구 생성)
```

**원칙**:
- 상위 문서는 하위 문서를 **참조만** 함
- 하위 문서는 상위 문서 내용을 **반복하지 않음**

### [Rule 3.2] 참조 링크 작성법

```markdown
<!-- ✅ Good: 명확한 링크 텍스트 -->
자세한 내용은 [개발 가이드라인](./docs/guidelines/development.md)을 참고하세요.

<!-- ❌ Bad: 모호한 링크 -->
[여기](./docs/guidelines/development.md)를 참고하세요.
```

## 4. 문서 갱신 원칙

### [Rule 4.1] 변경 이력 필수 기록

모든 문서는 하단에 변경 이력 포함:

```markdown
---

## 변경 이력

- **2025-11-28**: 중복 최소화 원칙 추가
- **2025-11-27**: 초기 작성
```

### [Rule 4.2] 연쇄 갱신 체크리스트

문서 수정 시 확인:
- [ ] 이 내용을 참조하는 다른 문서가 있는가?
- [ ] 참조하는 문서도 함께 업데이트했는가?
- [ ] 변경 이력을 기록했는가?

## 5. 문서 검증

### [Rule 5.1] 정기 중복 검사

분기별 실행:
```bash
# 중복 문구 검색 (예시)
grep -r "프로젝트 구조" docs/ --include="*.md" | sort | uniq -c
```

### [Rule 5.2] 문서 품질 체크리스트

작성 완료 시 확인:
- [ ] 다른 문서와 80% 이상 겹치지 않는가?
- [ ] 참조 링크가 모두 유효한가?
- [ ] 역할이 명확히 정의되어 있는가?
- [ ] 변경 이력이 기록되어 있는가?
- [ ] 코드 예제가 실행 가능한가?

## 7. 정량적 기준

### [Constraint 7.1] 중복 허용 한계

- **동일 문구 반복**: 2회 초과 금지
- **문서 내용 겹침**: 80% 초과 시 통합 필수
- **링크 깊이**: 3단계 이하 (README → docs → guidelines)

### [Constraint 7.2] 문서 길이 가이드

| 문서 유형 | 권장 길이 | 최대 길이 |
|-----------|-----------|-----------|
| README.md | 200줄 | 300줄 |
| QUICK_START.md | 300줄 | 500줄 |
| guidelines/*.md | 제한 없음 | - |

**초과 시 조치**: 별도 문서 분리 + 참조

## 8. 완료 조건 (Definition of Done)

문서 작성/수정 완료 전 자가 점검:

- [ ] 중복 검색 완료 (grep 실행)
- [ ] 참조 링크 모두 검증
- [ ] 변경 이력 기록
- [ ] 관련 문서 함께 갱신
- [ ] 체크리스트 100% 통과

## 부록: 중복 제거 사례

### Case 1: README.md vs QUICK_START.md

**문제**:
```
README.md (300줄)
QUICK_START.md (295줄)
→ 80% 중복
```

**해결**:
```bash
# 옵션 A: QUICK_START 삭제 (프로젝트 소규모)
rm QUICK_START.md

# 옵션 B: 역할 분리 (프로젝트 대규모)
# README.md: 50줄 (개요만)
# QUICK_START.md: 300줄 (상세 가이드)
```

### Case 2: 여러 문서에서 동일 설치 가이드 반복

**문제**:
```
README.md, INSTALL.md, docs/getting-started.md
→ 모두 "npm install" 설명
```

**해결**:
```markdown
<!-- README.md -->
## 설치
[설치 가이드](./INSTALL.md) 참고

<!-- docs/getting-started.md -->
## 환경 설정
설치는 [INSTALL.md](../INSTALL.md) 완료 후 진행
```
